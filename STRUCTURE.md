# Структура проекта

Проект реорганизован из папки `router2` в корневую папку `src` с декомпозицией по функциональным папкам.

## Структура папок

```
src/
├── pages/                      # Все страницы приложения
│   ├── Home/                   # Главная страница
│   │   ├── Home.jsx
│   │   └── Home.css
│   ├── Login/                  # Страница входа
│   │   ├── Login.jsx
│   │   └── Login.css
│   ├── Dashboard/              # Dashboard с вложенными страницами
│   │   ├── Dashboard.jsx
│   │   ├── Dashboard.css
│   │   ├── Profile/            # Профиль пользователя
│   │   │   ├── Profile.jsx
│   │   │   └── Profile.css
│   │   ├── Settings/           # Настройки
│   │   │   ├── Settings.jsx
│   │   │   └── Settings.css
│   │   └── Stats/              # Статистика
│   │       ├── Stats.jsx
│   │       └── Stats.css
│   ├── Characters/             # Страница списка персонажей
│   │   └── Characters.jsx
│   └── CharacterDetail/        # Детальная страница персонажа
│       └── CharacterDetail.jsx
│
├── components/                 # Переиспользуемые компоненты
│   └── ProtectedRoute/         # Компонент защиты маршрутов
│       └── ProtectedRoute.jsx
│
├── routes/                     # Конфигурация маршрутизации
│   ├── router.jsx              # Главный файл роутера
│   └── loaders.js              # Loaders для загрузки данных и защиты
│
├── data/                       # Статические данные
│   └── characters.json         # Данные персонажей
│
├── styles/                     # Глобальные и общие стили
│   ├── index.css               # Глобальные стили
│   ├── CategoryPage.css        # Стили для страницы категории
│   └── DetailPage.css          # Стили для детальной страницы
│
└── main.jsx                    # Точка входа приложения
```

## Основные изменения

1. **Декомпозиция по папкам**: Все файлы организованы по функциональному назначению
2. **Страницы в `pages/`**: Каждая страница имеет свою папку с компонентом и стилями
3. **Компоненты в `components/`**: Переиспользуемые компоненты вынесены отдельно
4. **Маршруты в `routes/`**: Конфигурация роутера и loaders отделены
5. **Данные в `data/`**: Статические JSON файлы
6. **Стили в `styles/`**: Глобальные и общие стили

## Как использовать

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

3. Откройте браузер и перейдите по адресу, указанному в терминале

## Примечания

- Все маршруты для страниц категории и детальной страницы защищены (требуют авторизации)
- Страница Login уже существует и работает корректно
- Все импорты обновлены под новую структуру
